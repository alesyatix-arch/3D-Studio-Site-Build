# Усиление сайта: фактура, эффекты, CTA, изображения

Сейчас сайт получился аккуратным, но "тихим": много воздуха, мало визуальных акцентов, единственная форма заявки только в подвале страниц. Делаем сайт более насыщенным, не ломая премиальный строгий стиль.

## 1. Фон страницы — белый с фактурой

Базовый `--background` остаётся белым. Добавляем три слоя фактуры в `src/styles.css`, которые работают глобально через `body::before` / утилитарные классы:

- **Тонкая сетка / "архитектурная миллиметровка"** — SVG-паттерн 1px линий с шагом 32px, цвет `--hairline`, opacity ~0.5. Создаёт ощущение чертежа, подходит для студии визуализации.
- **Зернистость (grain)** — лёгкий noise через SVG `feTurbulence`, opacity ~0.04. Убирает "цифровую плоскость", добавляет премиум-плёнку.
- **Радиальные тёплые пятна** — два больших `radial-gradient` с цветом `--accent` (бежевый) на 3–5% непрозрачности по углам секций. Перекликается с тёплым акцентом ТЗ.

Реализация:
- Глобальный grain + grid — на `body` (фиксированный, не скроллится).
- Утилитарный класс `.section-warm` — для отдельных секций с тёплым радиальным свечением.

## 2. Эффекты и микро-анимации

Подключаем уже доступные кейфреймы (`fade-in`, `scale-in`) + добавляем несколько новых утилит в `styles.css`:

- **Reveal on scroll** — обёртка-хук `useInView` (IntersectionObserver) + класс `.reveal` с переходом opacity/translateY. Применяем к заголовкам секций и карточкам.
- **Hover на карточках аудиторий / портфолио** — усилить: бежевая полоска-акцент снизу растёт слева направо, изображение слегка зумируется, появляется стрелка-ссылка.
- **Параллакс лёгкий** на hero-изображениях (translateY на скролл, ~10%).
- **Magnetic / underline-grow** для CTA-кнопок: подчёркивание из акцентного цвета растёт по hover.
- **Numbers count-up** в блоке статистики на `/about` (12+, 500+, 50+).
- **Sticky eyebrow** в длинных секциях — название секции прилипает слева при скролле (desktop).
- **Курсор-индикатор** на портфолио-тайлах: "View →" следует за курсором (только desktop).

Без тяжёлых библиотек: всё на CSS-transition + IntersectionObserver, без Framer Motion.

## 3. Больше CTA "Получить расчёт проекта"

Сейчас CTA только в финальной секции. Добавляем:

- **Sticky CTA в шапке** — справа в `Header.tsx` появляется компактная кнопка "Расчёт проекта" (видна на всех страницах, на mobile — иконка).
- **Inline CTA-баннер** — новый компонент `InlineCTA` (узкая полоса с текстом + кнопкой). Размещаем:
  - На главной: после блока "Аудитории" и после "Процесс".
  - На страницах аудиторий: после прайса и после процесса.
- **CTA в карточках аудиторий** — у каждой карточки (девелоперы / производители / дизайнеры) на главной добавить нижнюю строку "Рассчитать →".
- **Floating CTA на мобильном** — фиксированная кнопка снизу справа на mobile (скрывается при скролле вверх).
- **CTA в блоке прайса** — справа от таблицы цен кнопка "Получить расчёт под ваш объём".

Все CTA ведут либо на `/contacts`, либо открывают модальное окно с формой (вариант на выбор — см. вопросы ниже).

## 4. Подкрепление изображениями в блоках

Сейчас рендеры есть только в hero и портфолио. Добавляем:

- **Блок "Процесс"** на главной — рядом с каждым шагом (или фоном для активного шага) маленький render-thumbnail 1:1.
- **Блок "Ценности / почему мы"** — split-layout: текст слева, крупный render справа (sticky на scroll).
- **Аудитории на главной** — каждая из 3 карточек получает фоновое изображение (низкая opacity) + foreground текст.
- **Прайс-таблица** — слева от таблицы вертикальный render-коллаж (3 узких изображения).
- **About** — между статистикой и CTA добавить mosaic-сетку из 6 рендеров (галерея команды/работ).
- **Footer** — тонкая полоса-коллаж из миниатюр работ над основным содержимым футера.

Используем уже скачанные изображения из `src/assets/bg/` — новых ассетов не качаем в этой итерации.
